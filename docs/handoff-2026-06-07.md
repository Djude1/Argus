## 關鍵架構決策（已定案）

| 決策 | 結論 | 原因 |
|---|---|---|
| 攻擊平台 | **Kali Docker container**（加入 docker-compose） | 零衝突、工具齊全、快速部署 |
| 靶機 | **Mint 機**上跑 DVWA 或 Metasploitable 2 | 攻擊者/受害者分離，展示清楚 |
| 攻擊執行方式 | **Hermes-Agent**（`backend/apps/agent/`）呼叫 Kali | AI 自主決策攻擊鏈，是簡報亮點 |
| 資安程式碼位置 | 新功能寫進 `backend/apps/scans/security/`，舊程式碼不動 | 零遷移風險，邊界清楚 |
| Obsidian | 不引入為專案工具；個人可指向專案資料夾作 graph viewer | 現有 CLAUDE.md 架構已足夠 |

## 已知限制與注意事項

- **Windows build 禁用系統 Node**：`npm run build` 會 crash，一律用 `frontend/build-node22.ps1`
- **整合測試用 Docker**：`localhost:8080`，本機 runserver 缺少 Celery/Redis
- **Playwright 路徑**：必須設 `PLAYWRIGHT_BROWSERS_PATH=".ms-playwright"`
- **Kali 不要裝 Nuclei/Katana**：worker 已有，職責分離
- **掃描不能同時打同一目標**：Nuclei 完成後 Agent 才呼叫 Kali

- Kali Docker image 的 metasploit-framework 安裝方式（apt vs 官方安裝腳本）

- Python `ssl` 模組取得憑證資訊的 API

- OWASP Top 10（2021）A01~A10 的 CWE 編號對映表
