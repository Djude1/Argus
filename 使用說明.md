# Argus 使用說明

## 目前可用功能

- 使用者註冊與 JWT 登入。
- 建立授權式 ScanJob。
- 後端記錄授權證據：user、IP、timestamp、user-agent、授權網域。
- 明顯第三方或敏感網域需要二次確認。
- Passive / Active 模式檢查；Active 必須額外授權，並套用 RPS ≤ 2 速率限制。
- Playwright 同網域爬蟲：same-origin、最大深度、最大頁數、robots.txt、截圖、連結擷取、被阻擋頁面偵測（401/403/429）。
- 四維靜態掃描：SEO、AEO、GEO、被動資安，並含 GEO FAST 檢查（llms.txt、AI 爬蟲可存取性、JS 渲染依賴、語意化區塊、段落可引用性）。
- Findings、頁面資料、狀態查詢、截圖下載、Word 報告匯出。
- React 前台：註冊/登入、建立掃描、任務列表、互動報告（findings 分類/嚴重度篩選、截圖高光對齊）、複製 AI handoff prompt、匯出 Word。
- Django Admin：掃描總覽儀表、Agent token 監控、授權同意書唯讀檢視。

## 第一次啟動

1. 複製環境設定：

```powershell
Copy-Item .env.example .env
```

2. 編輯 `.env`：

- `DJANGO_SECRET_KEY` 換成本機開發用 secret。
- `JWT_SECRET_KEY` 換成本機開發用 secret。
- 本機快速測試可保留：
  - `CELERY_TASK_ALWAYS_EAGER=true`
  - `ARGUS_AUTO_QUEUE_SCANS=true`

3. 安裝 Python 依賴：

```powershell
uv sync
```

4. 安裝 Playwright Chromium：

```powershell
$env:PLAYWRIGHT_BROWSERS_PATH = ".ms-playwright"; uv run playwright install chromium
```

注意：`PLAYWRIGHT_BROWSERS_PATH=.ms-playwright` 會把 Chromium 安裝在專案內，避免污染使用者層級快取。

5. 建立資料庫：

```powershell
uv run python backend/manage.py migrate
```

6. 安裝前端依賴：

```powershell
cd frontend
npm.cmd install
cd ..
```

## 啟動開發環境

後端：

```powershell
uv run python backend/manage.py runserver 127.0.0.1:8000
```

前端：

```powershell
cd frontend
npm.cmd run dev
```

開啟：

```text
http://127.0.0.1:5173
```

## 使用流程

1. 在前端註冊帳號。
2. 登入。
3. 輸入你擁有或已取得書面授權的網址。
4. 勾選「我擁有此網站或已獲得書面授權測試」。
5. 若是第三方或敏感網域，必須再次確認授權。
6. 建立掃描。
7. 等待狀態更新為 `completed`。
8. 點選掃描任務查看 findings。
9. 點選 finding 可查看描述、修補方向，並複製 AI handoff prompt。
10. 點選「匯出 Word」下載報告。

## 驗證命令

後端：

```powershell
uv run python backend/manage.py check
uv run python backend/manage.py makemigrations --check --dry-run
uv run python backend/manage.py test apps.accounts apps.scans
uv run ruff check backend
```

前端：

```powershell
cd frontend
npm.cmd run build
```

## 注意事項

- 不要把 `.env`、`GoogleCloud_ApiKey.json`、API Key、Token、私鑰提交到版本控制。
- 只掃描你擁有或取得書面授權的網站。
- 主動式資安測試只在額外授權後使用。
- 本系統只提供問題描述、證據、修補方向與 AI handoff prompt，不輸出修復後程式碼。
