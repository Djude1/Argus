## 快速啟動

首次安裝完成後（依賴、migration、frontend build、Google Client ID 已設定），下次要用 Argus 時，**只要其中一種方式即可**：

### 模式 A：本機 runserver（最快）

```powershell
uv run python backend/manage.py runserver 127.0.0.1:8000
```

| 用途 | 網址 |
|---|---|
| 前台（Google 登入、掃描、報告） | http://127.0.0.1:8000/ |
| 公開介紹頁 | http://127.0.0.1:8000/project |
| 免費分析工具（不需登入） | http://127.0.0.1:8000/free-tools |
| React 管理後台（staff） | http://127.0.0.1:8000/admin/overview |
| Django Admin（superuser 後門） | http://127.0.0.1:8000/django-admin/ |
| API | http://127.0.0.1:8000/api/... |

### 模式 B：Docker 完整堆疊（web + worker + redis + postgres + nginx）

```powershell
docker compose up -d --build
```

| 用途 | 網址 |
|---|---|
| 前台 | http://localhost/ |
| React 管理後台 | http://localhost/admin/overview |
| Django Admin | http://localhost/django-admin/ |
| API | http://localhost/api/... |

### 兩種模式共用

- **一般使用者**：點「使用 Google 帳號登入」按鈕（首次自動建立帳號，並自動發 200 coin）。
- **系統管理員**：用 `createsuperuser` 建立的帳號，從 `/django-admin/` 登入，或在 React 前台登入後到 `/admin/overview`（需要 `is_staff=True`）。

### 修改了前端 (`frontend/src/*`) 之後

**模式 A 必須重 build 才看得到改動**：

```powershell
cd frontend ; .\build-node22.ps1 ; cd ..
```

> ⚠ 一律用 `build-node22.ps1`，**不可** `npm run build`。本機系統 Node v24 + Rollup 4.x 在 Windows 會 crash（`STATUS_STACK_BUFFER_OVERRUN`），必須走 portable Node v22 在 `D:\node22`。詳見 [CLAUDE.md](CLAUDE.md) Node 22 portable 段。

**模式 B**：`docker compose up -d --build frontend` 會自動 rebuild。

## 目前可用功能

- **登入**：Google OAuth 一鍵登入或註冊（首次自動建立帳號）；管理員以 `/django-admin/` 登入。
- **點數制度（取代舊月配額）**：每月登入自動發 200 coin；每頁掃描費 10 coin（預扣 `max_pages × 10`，完成後按實際頁數退差）；4 個購點方案（NT$100 / 450 / 800 / 1500）。
- **掃描任務**：建立授權式 ScanJob，後端記錄 user / IP / timestamp / user-agent / 授權網域；明顯第三方或敏感網域需要二次確認；Passive 預設、Active 需額外授權且 RPS ≤ 2。
- **爬蟲**：Playwright 同網域 BFS，最大深度 3、最多 50 頁；遵守 robots.txt；截圖、HTML、render 後 DOM、拓樸連結、被阻擋頁面偵測（401/403/429）。
- **四維掃描**：SEO / AEO / GEO / 被動資安，含 GEO FAST 檢查（llms.txt、AI 爬蟲可存取性、JS 渲染依賴、語意化區塊、段落可引用性）。
- **報告**：Word 報告（.docx）匯出、AI handoff prompt 複製、互動拓樸圖（ReactFlow）。
- **平台評論**：一人一則 + 討論 thread + 圖片上傳 + 「有幫助」點讚；管理員可回覆。
- **管理後台（React `/admin/*`）**：使用者管理 / 點數調整 / 交易紀錄 / 評論回覆 / 掃描監控 / CMS 內容編輯 / 操作紀錄（superuser 限定）。
- **公開免費工具 `/free-tools`**：測速 / 釣魚 URL 偵測 / 釣魚郵件偵測，AllowAny、不需登入、不扣點數。
- **真實 PWA**：可一鍵安裝到桌面 / 手機主畫面，支援離線 cache。

## 第一次安裝

1. 複製環境設定：

```powershell
Copy-Item .env.example .env
```

2. 編輯 `.env`：

- `DJANGO_SECRET_KEY` 換成 64-byte random（不可外流）。
- `JWT_SECRET_KEY` 換成另一段 64-byte random。
- `GOOGLE_OAUTH_CLIENT_ID` 從 Google Cloud Console 取得（見下節「Google OAuth 設定」）。
- 本機快速測試可保留：`CELERY_TASK_ALWAYS_EAGER=true`、`ARGUS_AUTO_QUEUE_SCANS=true`、`ARGUS_AGENT_ENABLED=false`。

3. 安裝 Python 依賴：

```powershell
uv sync
```

4. 安裝 Playwright Chromium（**必須在專案內，禁止污染全域**）：

```powershell
$env:PLAYWRIGHT_BROWSERS_PATH = ".ms-playwright"; uv run playwright install chromium
```

5. 建立資料庫（含自動 seed：PricingPlan、CMS 內容）：

```powershell
uv run python backend/manage.py migrate
```

6. 建立 superuser：

```powershell
uv run python backend/manage.py createsuperuser
```

7. 安裝前端依賴並 build：

```powershell
cd frontend
D:\node22\npm.cmd install
.\build-node22.ps1
cd ..
```

8. 啟動：

```powershell
uv run python backend/manage.py runserver 127.0.0.1:8000
```

## 進階：前端 hot-reload（改 React 不必每次 build）

後端：

```powershell
uv run python backend/manage.py runserver 127.0.0.1:8000
```

前端 dev server（另開一個終端機）：

```powershell
cd frontend
D:\node22\npm.cmd run dev
```

打開 http://127.0.0.1:5173/（Vite dev server）；dev server 會 proxy API 請求到 backend 8000。

> 注意：一般使用情境用「快速啟動」模式 A 就好，hot-reload 模式只在開發前端時才需要。

## Google OAuth 設定（首次部署必做）

一般使用者透過 Google 登入，必須先取得 Google Cloud OAuth Client ID：

1. 進入 [Google Cloud Console](https://console.cloud.google.com/) 並選擇或建立一個專案。
2. 至「APIs & Services > OAuth consent screen」，設定使用者類型（External）與應用程式名稱。
3. 至「APIs & Services > Credentials > Create credentials > OAuth client ID」。
4. Application type 選 **Web application**。
5. Authorized JavaScript origins 加入：
   - `http://localhost:5173`（本機 Vite dev）
   - `http://localhost`（Docker compose 透過 nginx）
   - `http://127.0.0.1:8000`（本機 runserver 模式）
6. 建立後取得 Client ID（形如 `xxx.apps.googleusercontent.com`），填入 `.env`：

```text
GOOGLE_OAUTH_CLIENT_ID=xxx.apps.googleusercontent.com
```

7. 重新啟動 backend；登入頁就會出現 Google 登入按鈕。

未設定 `GOOGLE_OAUTH_CLIENT_ID` 時，前端會顯示「尚未設定」訊息，後端 `/api/auth/google/` 會回 HTTP 503。

## 使用流程（一般使用者）

1. 開啟前端網址（模式 A：http://127.0.0.1:8000/ 或模式 B：http://localhost/）。
2. 點選「使用 Google 帳號登入」（首次自動建立帳號，並自動發 200 coin）。
3. 輸入你**擁有**或**已取得書面授權**的網址。
4. 勾選授權同意書「我擁有此網站或已獲得書面授權測試」。
5. 若是第三方或敏感網域（如 google.com、銀行等），系統會要求再次確認。
6. 確認預扣 coin 後建立掃描；點數不足會擋下並提示購點。
7. 等待狀態進入 `completed`（過程中可看 progress JSON 即時進度，可隨時 `cancel/` 自動退款）。
8. 點選掃描任務查看互動報告（findings 列表 + 截圖高光對齊）。
9. 點選 finding 可查看描述、修補方向，並複製 AI handoff prompt 貼到 ChatGPT / Claude 詢問修復方向。
10. 點選「下載 Word 報告」取得 `.docx`。

## Cache replay 重跑掃描（不重新打目標站）

修改 scanner 邏輯後，可以不向目標網站重發請求，直接用已儲存的 Page 內容重跑：

```powershell
uv run python backend/manage.py rerun_scan <scan_id>
```

加 `--keep-findings` 保留原 findings 僅追加新項。被阻擋（401/403/429）的頁面與站台層級 GEO FAST 訊號（llms.txt、AI 爬蟲）不參與 replay。

## Docker 細節

完整端到端容器化（web/worker/redis/db/nginx），所有依賴在 image 與 volume 內，不污染主機：

```powershell
docker compose up -d --build
```

啟動後：

- 前端 SPA + React Router：`http://localhost/`
- React 管理後台：`http://localhost/admin/overview`
- Django Admin：`http://localhost/django-admin/`
- API：`http://localhost/api/`

說明：

- 本 compose 為本機開發配置；`DJANGO_DEBUG=true` 讓 runserver 自動服務 Admin 靜態檔。
- 正式部署需改用 gunicorn + `collectstatic` 並設 `DJANGO_DEBUG=false`。
- 容器內 Playwright Chromium 位於 `/ms-playwright`（與主機 `.ms-playwright` 各自獨立）。
- 資料庫資料持久化於 docker volume `postgres_data`，截圖與報告於 `media_data`。

## 注意事項

- **絕不提交**：`.env`、`GoogleCloud_ApiKey.json`、API Key、Token、私鑰到版本控制（已在 `.gitignore`）。
- **只掃描**你擁有或取得書面授權的網站。
- **主動式資安測試**只在使用者額外勾選授權後使用，RPS ≤ 2。
- 本系統只提供問題描述、證據、修補方向與 AI handoff prompt，**不輸出修復後程式碼**。
- Python 套件一律用 `uv add` / `uv run`，禁止 `pip install` 污染全域。
- Node 套件一律安裝在 `frontend/node_modules`，禁止 `npm install -g` 污染全域。
- Playwright 瀏覽器一律放 `.ms-playwright`，禁止寫入 `%USERPROFILE%\AppData\Local\ms-playwright`。
