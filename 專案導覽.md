# 專案導覽 · SKILL 與子目錄索引

> 給 Claude / Codex / 新接手者的「地圖」：一眼看出有哪些 skill、何時用、有哪些子目錄規則、各自管什麼。
> **鐵則**：每次對話若使用者講了新規則 / 決策 / 地雷，**當次就要更新對應 skill 或 CLAUDE.md**，保持最新版本——別只留在對話記憶裡（對話一結束就忘）。詳見最後一節。

---

## 一、規則載入分層（先懂這個）

| 層 | 檔案 | 載入時機 | 可靠度 |
|---|---|---|---|
| 使用者層 | `~/.claude/CLAUDE.md` | 每 session 必載（全域） | ★★★ |
| 專案層 | `CLAUDE.md` | 每 session 必載 | ★★★ |
| 子目錄層 | `frontend/CLAUDE.md`、`backend/apps/*/CLAUDE.md` | 進該目錄工作時 | ★★ |
| 個人覆寫層 | `CLAUDE.local.md` | 每 session 必載（不提交、不被 pull 蓋） | ★★★ |
| Claude skill | `.claude/skills/*/SKILL.md` | description 命中才載 body | ★★ |
| Codex skill | `skills/argus-project/` | Codex 用 | — |

→ 「永遠要記得」放 CLAUDE.md / CLAUDE.local.md；「特定情境才需要」放 skill。

---

## 二、SKILL 索引（何時用哪支）

| Skill | 觸發時機 | 位置 | 給誰 |
|---|---|---|---|
| `argus-ui-design` | 任何前端 UI/UX：App.jsx 頁面/元件、styles.css、前台/後台、按鈕/導覽/動畫/配色/版面/「美化」 | `.claude/skills/argus-ui-design/SKILL.md` | Claude（自動載入） |
| `argus-git-safety` | 任何 `git add`/`commit`/`push`、部署/上線/共用 repo/組員協作 | `.claude/skills/argus-git-safety/SKILL.md` | Claude（自動載入） |
| `argus-project` | Argus 總規則、環境隔離、API provider、交接 | `skills/argus-project/SKILL.md`（+ `references/`） | Codex（Claude 不自動載入，可手動參考） |

---

## 三、子目錄 CLAUDE.md 索引（各管什麼）

| 路徑 | 模組 | 重點 |
|---|---|---|
| `frontend/CLAUDE.md` | 前端 | App.jsx 6500+ 行單檔、build-node22、樣式/狀態規範 |
| `backend/apps/accounts/CLAUDE.md` | 帳號 | User(AbstractUser)、JWT、Google 登入、不簽發 staff |
| `backend/apps/scans/CLAUDE.md` | 掃描（核心） | ScanJob 狀態機、Playwright、合作式取消 |
| `backend/apps/agent/CLAUDE.md` | Agent | Hermes-Agent（預設關閉）、provider chain、same-origin、禁印 key |
| `backend/apps/billing/CLAUDE.md` | 點數 | `services.py` 是 wallet 唯一寫入入口 |
| `backend/apps/reviews/CLAUDE.md` | 評論 | 一人一則(OneToOne)+thread+圖片+點讚 |
| `backend/apps/admin_api/CLAUDE.md` | 後台 API | IsAdminUser、AdminAuditLog 稽核、調點數走 billing.services |
| `backend/apps/content/CLAUDE.md` | CMS | 公開唯讀；寫入走 admin_api/cms |
| `backend/apps/insights/CLAUDE.md` | 免費工具 | `/free-tools`、SSRF 防護（PublicHostError 不可移除） |

> 共 8 個 Django app；billing / scans 是最複雜、最該先讀的兩個。

---

## 四、何時該主動讀 / 用什麼

- 改**前端任何畫面** → 自動載入 `argus-ui-design` 再動手。
- 要 **commit / push / 談部署** → 自動載入 `argus-git-safety` 再動手。
- 動**某個 backend app** → 先讀該 app 的 `CLAUDE.md`（再讀 `scans` 等核心 app 的話必讀其狀態機規則）。
- 不確定現況 → 先讀 `CLAUDE.md`、`ONBOARDING.md`、本檔，再動手（禁止猜測）。

---

## 五、每次對話即時更新（防遺忘鐵則）

1. **使用者在對話中講的新規則 / 決策 / 地雷，當次就要落地**到對應檔案（skill / `CLAUDE.md` / 子目錄 `CLAUDE.md` / 本檔），不要只留在對話記憶——對話結束就忘，下個 Claude 會重蹈覆轍。
2. **新增 / 改 skill** → 同步 `CLAUDE.md` 的「Skills 表格」與本檔第二節（「文件同步強制規則」要求）。
3. **新增 / 改子目錄規則** → 同步本檔第三節。
4. **機器專屬設定**（測試機、RTK 路徑等）→ 只寫 `CLAUDE.local.md`，不寫進會被 push 的檔。
5. 改完跑一次自我核對：本檔三張表 ↔ 實際檔案是否一致（缺漏即補）。
